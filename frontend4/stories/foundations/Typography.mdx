## Naming convention

Class names describe **visual size**, not HTML semantics. `type-title-xl` is
the largest title regardless of whether it sits inside an `<h1>`, `<h2>`, or a
plain `<div>`.

- **Pick the HTML element from the document outline** — heading order, semantic
  meaning, accessibility tree.
- **Pick the type class from the visual size you want.**
- The two are independent decisions.

This means an `<h2 className="type-title-xl">` is fine when a sub-page has a
secondary heading you want to be visually dominant, and an
`<h1 className="type-title-md">` is fine for a quieter sub-page hero.

The scale has four tiers:

| Tier | Classes | Typeface | Purpose |
|---|---|---|---|
| Title | `type-title-xl`, `type-title-lg`, `type-title-md`, `type-title-sm`, `type-title-xs` | Josefin Sans 700 | Display headings |
| Body | `type-body-lg`, `type-body`, `type-caption` | Nunito Sans 400 | Reading text |
| Eyebrow | `type-eyebrow` | Josefin Sans 700 | All-caps brand-color label |
| Micro | `type-micro-lg`, `type-micro` | Josefin Sans 700 | Uppercase tracked UI labels |

## When to use what

The type scale is small on purpose — pick the *visual size* you want, then
pick the HTML element that matches the document outline. Most page-level
patterns look like one of these:

### Common compositions

- **Page hero** — `type-eyebrow` + `type-title-xl` + `type-body-lg`
- **Section header** — `type-eyebrow` + `type-title-lg` + `type-body`
- **Card** — `type-title-sm` + `type-body` + `type-micro` link
- **Footer / metadata** — `type-micro` heading + `type-caption` items
- **Form field** — `type-micro` label + input
- **Badge / chip** — `type-micro` text on a colored background

A live "in context" composition is on the **In Context** sub-story.

### Right vs wrong

The class describes visual size. The element describes the document outline.

✅ **Right** — independent decisions:

```tsx
// Sub-page hero — visually dominant but second in document order.
<h1 className="type-title-md">Privacy Policy</h1>

// A bold section title that happens to be an h2.
<h2 className="type-title-xl">Why choose laser cleaning?</h2>
```

❌ **Wrong** — using a title size for non-heading content:

```tsx
// A button label is not a title. Reach for type-micro.
<div className="type-title-xl">Submit</div>
```

❌ **Wrong** — using HTML element name in the class name:

```tsx
// type-h2 is gone. There's no class named after the tag.
<h2 className="type-h2">Section</h2>
```

### Default element when using the Typography wrapper

When using the `Typography` component (`app/components/Typography.tsx`),
each variant has a sensible default tag. Override with `as`:

| Variant | Default element |
|---|---|
| `title-xl` | `<h1>` |
| `title-lg` | `<h2>` |
| `title-md` | `<h3>` |
| `title-sm` | `<h4>` |
| `body-lg`, `body`, `caption` | `<p>` |
| `eyebrow`, `micro-lg`, `micro` | `<span>` |

```tsx
// Override the default tag when the outline calls for it:
<Typography variant="title-md" as="h1">Sub-page Hero</Typography>
```
