## How colors are wired up

`globals.css` defines color values as CSS custom properties on `:root`,
then the `@theme inline` block re-exports them with the `--color-` prefix
that Tailwind v4 reads to generate utility classes:

```css
:root {
  --brand: #D4834A;
}

@theme inline {
  --color-brand: var(--brand);   /* generates bg-brand, text-brand, border-brand */
}
```

That double layer means **the source of truth is always the bare variable
name** (`--brand`), and **Tailwind classes are derived** (`bg-brand`,
`text-brand`, `border-brand`). To repaint the site for a client review,
swap the `:root` block — never edit the `@theme inline` block.

The token vocabulary is intentionally small (11 colors, no per-component
shades). If you need a tint or shade in one place, reach for an opacity
modifier (`bg-brand/10`, `text-foreground/70`) instead of adding a new token.

### How to swap a preset

In `app/globals.css`, the active preset sits inside `:root`. The other two
are kept as commented-out blocks directly below. To switch:

1. Comment out the current `:root { ... }` block.
2. Uncomment the preset you want to try.
3. Reload — every component re-paints, no other code changes needed.

```css
/* Active */
:root {
  --background: #141311;
  /* ...rest of Preset A... */
}

/* Inactive — uncomment to try */
/*
:root {
  --background: #faf8f4;
  ...
}
*/
```

Don't define new color tokens during a preset experiment — if a preset
needs a color the others don't have, the system breaks. Stick to the 11
shared tokens.

## Don'ts

- ❌ **Don't add hex values directly in components.** Every color in the
  app should resolve through a CSS variable. Hex literals in className
  arbitrary values (`bg-[#ff0000]`) bypass the preset system.
- ❌ **Don't introduce new color tokens unilaterally.** If you need a new
  shade, first ask whether opacity modifiers (`bg-brand/10`,
  `text-foreground/70`) cover the case. They usually do.
- ❌ **Don't assume `--secondary` and `--foreground` are the same.** They
  match in Preset A but diverge in B and C. Always pick the token that
  describes intent.
