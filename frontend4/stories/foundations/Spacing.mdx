## The base unit

```
1 spacing unit = 0.25rem = 4px (at the default 16px root font size)
```

A class like `p-4` resolves to `padding: 1rem` (4 × 0.25rem = 16px). The
scale skips up in 0.25rem steps from 0 to 12, then doubles in pace as the
values get larger (14, 16, 20, 24, … 96). This keeps the scale finite —
36 tokens cover the full range from a single pixel to 384px.

This codebase **does not customise the spacing scale**. There's no `--spacing`
override and no extra entries in `@theme`. What you see below is what
Tailwind ships.

## How tokens map to utilities

Most utilities take a token name and apply it as a property:

```tsx
<div className="p-4">          {/* padding: 1rem  */}
<div className="px-8 py-12">    {/* horizontal 2rem, vertical 3rem */}
<div className="mt-24 mb-16">   {/* margin-top 6rem, margin-bottom 4rem */}
<div className="gap-9">         {/* gap: 2.25rem (used in nav row) */}
<div className="space-y-10">    {/* vertical rhythm — 2.5rem between children */}
```

Negative spacing works too: `-mt-4` is `margin-top: -1rem`. Fractional
tokens (`0.5`, `1.5`, `2.5`, `3.5`) cover the small adjustments
typography sometimes needs.

### Common patterns in this codebase

| Pattern | Token | Where used |
|---|---|---|
| Page horizontal padding | `px-8` (2rem · 32px) | Every `mx-auto max-w-[1280px]` container |
| Section vertical rhythm | `pt-24 pb-12` or `py-20` | Section headers and footers |
| Component internal padding | `p-6` to `p-8` (1.5–2rem) | Cards, accordion answers |
| Form field gap | `space-y-5` (1.25rem) | QuoteForm field stack |
| Header nav link gap | `gap-9` (2.25rem) | Header desktop nav |
| Footer column gap | `gap-10` (2.5rem) | Footer 4-column grid |

## Arbitrary values

Tailwind v4 supports `[bracket]` syntax for one-off values that don't fit
the scale. The codebase uses these where a design constraint is genuinely
arbitrary — not as a way to skip thinking about the scale.

### When to use a scale token vs an arbitrary value

✅ **Use a scale token** when you're laying out something that wants to
breathe with the rest of the page — section padding, card spacing,
typographic rhythm. The scale is what gives the site its consistent
vertical music.

✅ **Use `[arbitrary]`** when a value is fixed by an external constraint —
a container width that matches a specific breakpoint, a header offset
that compensates for a fixed bar, an image dimension that matches an
aspect ratio. These are exceptions, not defaults.

❌ **Don't reach for `[18px]` because `p-5` (20px) is "too much".** That's
the scale doing its job — pulling the design toward a coherent rhythm.
The right answer is usually `p-4` or `p-5`, not `p-[18px]`.

## Tailwind v4 specifics

Two things changed in v4 that affect spacing:

1. **No `tailwind.config.ts` extension.** The scale lives in Tailwind's
   defaults — there's no config file in this project to override or
   extend. To add a spacing token, set `--spacing-*` inside
   `@theme inline` in `globals.css`.
2. **The `--spacing` base.** v4 derives the entire scale from a single
   `--spacing` variable. Override it once and every token rescales:

   ```css
   @theme inline {
     --spacing: 0.2rem;  /* would shrink the whole scale to 80% */
   }
   ```

   Don't do that here — the design was tuned for the default 4px base.
